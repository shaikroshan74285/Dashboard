Implemented a full frontend-first MarketPulse India dashboard, plus an optional Python proxy backend for secure Sarvam key usage.

What I built

Complete white-themed, data-dense dashboard UI with sections for RSS intelligence, sentiment pulse, risk radar, TradingView market widgets, AI Q&A, translation, TTS voice output, and YouTube live wall.
India-focused default RSS source pack (ET, Livemint, Business Standard, SEBI, RBI, PIB, Google News India market query) with auto-refresh and custom RSS add/remove.
AI legal safety layer:
Prompt guardrails (analysis only, no direct trade calls)
Output compliance blocking for direct buy/sell/target/stop-loss style directives.
3-API flow support:
Chat API
Translate API
TTS API
Plus browser speech fallback if TTS API fails.
Optional FastAPI backend proxy (/api/chat, /api/translate, /api/tts, /api/rss) for production-safe key handling.

Validation done

node --check app.js passed.
python -m py_compile backend/app.py passed.
Run

Frontend-only:
python -m http.server 5500
Open http://localhost:5500
Configure API settings in UI.
Optional backend:
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env and set SARVAM_API_KEY
uvicorn app:app --reload --port 8000
In UI, switch mode to Backend Proxy.
